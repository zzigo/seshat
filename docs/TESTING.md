# Testing

## Test frameworks

Seshat uses two lightweight test stacks:

- Node.js built-in `node:test`, executed from TypeScript through `tsx`.
- Python standard-library `unittest` for the ingestion pipeline.

Astro and TypeScript correctness are checked separately through `astro check` and `tsc --noEmit`.

## Full verification

Run from the repository root:

```bash
npm test
npm run typecheck
npm run build
```

`npm test` builds reusable packages before running tests, so stale package output cannot hide type/API mismatches.

## Writing tests

TypeScript tests use `*.test.ts` in each package's `test/` directory:

```ts
import assert from 'node:assert/strict';
import test from 'node:test';

test('describes one behavior', () => {
  assert.equal(actual, expected);
});
```

Python tests use `test_*.py`, `unittest.TestCase`, temporary directories and injected fake converters/chunkers. Prefer dependency injection over loading real Docling models in unit tests.

When changing ingestion output, verify both the returned manifest and generated bytes/files. When changing catalog queue semantics, test stage order, initial status and idempotence.

## Integration checks for production-facing changes

No automated integration suite currently covers PostgreSQL, R2, Authentik, Dockview or Handsontable. For changes in those areas, add proportionate manual checks:

### Upload/worker

1. Drop a small non-sensitive document.
2. Confirm a table row appears without navigation.
3. Observe HUD transitions from upload to extraction to identification.
4. Open original, text and structure pods.
5. Confirm PostgreSQL job states and R2 artifacts if debugging.

### Metadata

1. Edit a field in Handsontable.
2. Wait for the save indicator.
3. Reload and confirm persistence.
4. Requeue identification and confirm manually curated fields remain unchanged.

### Deletion

Use a disposable test document only. Click the leading `×`, then confirm the row and pods disappear and the reference prefix is absent in R2. Deletion is intentionally immediate and has no confirmation dialog.

### Authentication

Check that protected pages redirect to `/login` and protected APIs return `401` without a session. Test actual OIDC login in a browser; never automate by copying production session cookies into fixtures.

No GitHub Actions or other CI workflow is committed. Local verification is therefore the release gate. Adding CI for `npm ci`, `npm test`, `npm run typecheck`, `npm run build` and the Python version matrix is a high-value next step.
